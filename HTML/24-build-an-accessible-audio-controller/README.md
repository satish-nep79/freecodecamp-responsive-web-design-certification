# Build an Accessible Audio Controller

This project is part of the freeCodeCamp Responsive Web Design certification.

The goal of this project was to create an accessible audio control interface using semantic HTML and ARIA attributes to improve usability for all users.

## Features
- Semantic HTML structure
- Accessible buttons for Play and Mute actions
- Volume control using `input type="range"`
- ARIA labeling with `aria-labelledby`
- Clear descriptive labels for assistive technologies
- Simple and user-friendly control layout

## Technologies Used
- HTML5

## What I Learned
- How to build accessible UI controls using HTML
- How to use ARIA attributes to improve accessibility
- How to structure interactive elements semantically
- How to label range inputs for screen readers
- How small accessibility improvements improve usability

## Controls Included
- Play button
- Volume slider
- Mute button

## HTML Concepts Practiced
- Semantic button elements
- Input type="range" for volume control
- ARIA attributes for accessibility
- aria-labelledby for descriptive labels
- Accessible form controls
- Screen reader support
- Keyboard navigation
- Accessible user interface design
